## 12. 上线执行准入门槛

### Go 条件

- 所有必填生产环境变量已确认
- 发布镜像 tag 已确认
- 回滚镜像 tag 已确认
- 数据库备份已完成
- 文件备份已完成
- 负责人已确认
- 上线窗口已确认
- Final Go 已完成
- `release-smoke` 通过
- 回滚方案已确认

### No-Go 条件

- 任一真实密钥缺失
- 回滚 tag 缺失
- 数据库备份缺失
- 文件备份缺失
- `/ready` 不通过
- 登录不通过
- 认证 mock 被误启用
- 文件存储不可用
- 幂等重复写入
- 回滚负责人未确认
- 发现真实密钥被写入仓库

## 16. 最终结论模板

### Go

适用于所有必填项已确认、备份与回滚准备就绪、Final Go 已完成、上线窗口和人员已确认的情况。

### Conditional Go

适用于存在少量非阻断待确认项，但不影响正式发布执行的情况，需由发布负责人明确条件和补充时限。

### No-Go

适用于存在阻断性问题，或任一关键参数、备份、回滚、权限、密钥确认未完成的情况。
