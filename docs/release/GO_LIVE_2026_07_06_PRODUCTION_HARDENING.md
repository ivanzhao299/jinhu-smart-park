# 2026-07-06 生产上线加固与决策权限计划

## 上线目标

金湖 Smart Park 计划在 2026-07-06（周一）启动生产使用。本轮加固目标不是继续扩展新大模块，而是让现有系统满足真实生产启动的最低要求：

1. 管理员能够看到全部已上线模块。
2. 现场人员能够从移动端进入自己的作业终端。
3. 工程、物业、安全、招商、财务等角色的查看、填报、审批和决策权限边界明确。
4. 工程管理、巡检、隐患、工单、组织、用户、角色、字典等核心页面没有明显死按钮和空下拉。
5. 上线前能够用命令快速检查关键入口、迁移和权限矩阵是否齐备。

## 本轮已加固内容

### 1. 决策权限迁移

新增迁移：

`database/migrations/000166_go_live_decision_permission_hardening.sql`

该迁移只维护 `rel_role_perm`，不改业务表结构。覆盖真实职责分工中的 `JH_*` 角色，重点修复“角色存在但工程管理/决策权限没有落到生产权限表”的问题。

### 2. 上线检查脚本

新增命令：

```bash
pnpm go-live:check
```

检查范围：

1. 移动端现场作业入口是否存在。
2. 工程移动终端、工程项目、计划、日报、巡检、整改、验收入口是否存在。
3. 菜单配置是否包含关键上线页面。
4. 工程 RBAC、管理员可见性、隐患字典、上线决策权限迁移是否存在。
5. 典型死按钮/占位按钮模式是否残留。

该脚本不读取生产密码、不连接数据库，适合作为上线包的本地静态 gate。

2. **管理员 UAT**：管理员登录后必须能看到：总览、资产、招商、IoT、能耗、机器人、视频、工单、安全、工程、系统。

## 周一上线操作原则

1. 先开通管理员和业务负责人账号，不一次性放开全部普通用户。
2. 所有真实操作先从“查看、填报、提交、审核”四类动作验证。
3. 财务结算、服务器配置、生产运维类高风险动作暂不进入首日生产闭环。
4. 当天每个角色必须记录至少一条 UAT 结论：PASS / BLOCKED / NEEDS_FIX。
5. 出现权限缺口时优先补权限迁移，不临时在前端绕过权限。
