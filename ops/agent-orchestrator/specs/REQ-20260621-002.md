# REQ-20260621-002: Trial Launch Production Evidence Automation Queue

## Background

The orchestrator now supports a file-driven task pool, centralized dispatch dry-runs, result auditing, and one-click validation. The previous trial-launch batch closed several documentation and evidence tasks, but the release remains No-Go because production-use evidence is not closed for rollback readiness and the target release chain.

Agent 5's rollback evidence concluded that the scripts exist, but target-environment evidence is still missing for:

- production deployment pre-checks;
- database migration verification;
- production seed verification;
- initialization baseline checks;
- bootstrap-admin checks;
- production health checks;
- Docker container login verification;
- file upload, download, and delete smoke;
- database backup;
- file backup;
- rollback image/tag evidence;
- Docker cleanup;
- a consolidated release gate report with Go / Conditional-Go / No-Go judgment.

## User Goal

Move Jinhu Smart Park toward a trial-launch-ready state by automating and scripting production-readiness evidence collection. The next step is not business feature development. It is to prepare auditable, repeatable, and rollback-aware release evidence tasks that agents can claim through the orchestrator queue.

## Requirements

1. Write the natural-language request into `ops/agent-orchestrator/intake/current-request.md`.
2. Generate this formal REQ document.
3. Generate a technical decomposition document.
4. Generate a machine-readable task queue that `orchestratorctl` and `dispatch-ready-agents.mjs` can use.
5. Generate the next task batch with 5-8 READY tasks.
6. Preserve existing DONE tasks and historical results.
7. Keep `task-locks.json` without active locks.
8. Do not create fake results for new tasks.
9. Add a production evidence plan under `docs/release/trial-launch-production-evidence-plan.md`.
10. Do not execute real production deployment, production environment connection, migration, seed, cleanup, reset, or destructive commands in this turn.
