# 民宿 MVP 规划

## Goal

在共享房产底座上建立面向自营民宿运营人员和现场管家的整套房源短租闭环，覆盖价格、房态、预订、实名入住、退房、收退款和保洁周转。

## Dependencies

- 依赖 `07-24-scope-uat-alignment`。
- 依赖 `07-24-shared-property-foundation` 的经营模式和统一占用能力。
- 实施优先于住房出租 MVP。

## Users

- 民宿运营负责人
- 前台/订单运营
- 现场管家
- 保洁人员
- 维修人员
- 财务人员
- 审计/管理员

首期不提供住客公开预订或自助端。

## Core Flow

1. 将整套经营房源切换为短租模式。
2. 配置基础日价、日期覆盖和简单取消规则。
3. 查询日期可用性并创建人工/直订预订。
4. 确认订单并锁定 `[入住日, 退房日)` 库存。
5. 登记实收、押金或欠款。
6. 登记一位或多位实际入住人，完成内部实名核验。
7. 人工发放钥匙、门卡或入住凭证并办理入住。
8. 办理续住、改期、换房（后续可选）或退房。
9. 人工确认退款、扣费和赔偿。
10. 退房自动进入待保洁并创建保洁任务。
11. 保洁/检查完成后恢复可售，异常转维修或赔偿。

## MVP Scope

### 房态与库存

- 按整套房源、按入住夜管理可用性。
- 支持可售、临时锁定、已预订、在住、待保洁、保洁中、待检查、维修占用、停用。
- 预订确认、取消、改期、入住和退房必须同步统一占用账本。
- 临时锁定具有过期时间，避免长期占用库存。

### 价格

- 基础日价。
- 按日期覆盖价格。
- 订单逐日价格快照和总价。
- 改期重新计算差价并保留调整记录。
- 暂不支持会员价、优惠券、套餐、连住折扣、渠道价和收益管理。

### 预订

- 自营直订和人工录单。
- 记录订单来源、渠道名称、外部渠道订单号和同步状态。
- 暂不对接 OTA 的库存、价格和订单同步。
- 简单取消规则：免费取消截止时间，超时固定金额或比例取消费。
- 订单保存取消规则快照。

### 入住与隐私

- 一笔订单支持多位实际入住人。
- 记录姓名、证件类型、证件号码、手机号、实名核验状态和操作人。
- 证件信息加密或脱敏展示并受独立权限控制。
- 首期只做内部实名登记，不接公安住宿登记系统。

### 凭证与门锁

- 人工管理钥匙、门卡和入住凭证。
- 记录发放、领取、回收和操作审计。
- 不保存明文长期门锁密码。
- 预留门锁设备 ID 和临时密码任务状态，不接智能门锁。

### 财务

- 形成民宿订单账单、押金、附加费、损坏赔偿、实收和退款记录。
- 只支持人工收款登记、核销和人工退款确认。
- 预留支付渠道和流水号，不接在线支付网关。
- 特殊减免必须具备权限、原因和审计。

### 保洁周转

- 退房自动创建保洁任务并将房态置为待保洁。
- 移动端执行、拍照、登记耗材和上报异常。
- 完成后按项目配置进入待检查或恢复可售。
- 损坏可转维修工单或赔偿待处理项。

## Acceptance Criteria

- [ ] 可从空闲房源完成预订、确认、入住、退房和恢复可售闭环。
- [ ] 同一房源同一夜不能存在两个有效订单或长租占用。
- [ ] 订单金额可追溯到每日价格快照。
- [ ] 取消、改期、扣费、退款和减免均有审计记录。
- [ ] 多入住人实名信息按敏感数据保护。
- [ ] 退房不会直接恢复可售，必须经过保洁/检查规则。
- [ ] 管理后台和 390px 级现场移动端可完成核心操作。

## Out of Scope

- OTA 实时对接。
- 在线支付和自动退款。
- 智能门锁。
- 公安住宿登记接口。
- 公开预订网站、App、小程序和住客自助入住。
- 房间/床位库存。
- 会员、优惠券、套餐和复杂收益管理。
- 复杂保洁排班、路线、计件工资和机器人派单。
