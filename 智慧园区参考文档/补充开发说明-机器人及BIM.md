# 一、项目总体定位

本项目定位：

“产业园数字运营 SaaS 平台”。

平台核心方向：

- 智慧园区
- IoT
- BIM
- 数字孪生
- 能源管理
- AI运营
- 机器人运营
- 视频AI
- 工单运维
- SaaS平台

# 二、平台架构总体原则

## 2.1 平台核心原则

平台本质：

不是单园区定制项目。

而是：

“标准化 SaaS 产品”。

因此：

所有功能必须模块化。

禁止项目制硬编码。

# 三、SaaS总体架构要求

## 3.1 多租户架构（必须）

系统必须支持：

平台
├── 租户A
├── 租户B
└── 租户C

所有数据必须 tenant_id 隔离。

## 3.2 多园区支持

一个租户支持多个园区。

## 3.3 RBAC权限体系

必须支持：

- 多角色
- 多岗位
- 权限叠加
- 数据隔离
- 模块授权

# 八、BIM与数字孪生体系

## BIM技术路线

Revit
→ IFC
→ IFC.js
→ xeokit
→ Three.js

## IFC.js

负责：

- IFC解析
- GUID解析
- 属性解析
- 空间树解析

## xeokit

负责：

- BIM加载
- BIM性能优化
- 构件选择
- 空间树
- 属性系统

## Three.js

负责：

- WebGL渲染
- Mesh
- 光照
- 摄像机
