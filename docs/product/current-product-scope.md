# 当前产品范围与分批开放策略

> 状态日期：2026-07-24
> 本文件是当前产品范围的权威入口。历史文档中的“首发范围”“二期范围”“暂缓开放”和“生产开放”保留其阶段性语义，但不再定义产品最终边界。

## 1. 当前阶段

JinHu Smart Park 仍处于持续开发、功能完善和 UAT 验收阶段，尚未真实投入生产运营。

当前部署使用生产相似配置和部分 `production` 技术命名，用于验证未来正式投产所需的部署、迁移、认证、权限、财务和运维能力。该环境属于 UAT，不等同于真实生产环境。

## 2. 产品范围原则

- 当前已经设计开发的全部功能均属于计划上线范围。
- 不再以历史菜单白名单区分永久“一期产品”和“二期产品”。
- 允许各模块分批开发完善、分批 UAT、分批正式开放。
- 首次真实投产只开放已通过当前版本 UAT 且具备运维保障的模块。
- 尚未开放的模块仍属于目标范围，不得因为菜单隐藏而从路线图、质量治理或安全检查中移除。
- 菜单、权限、模块授权和功能开关只负责执行阶段性开放策略。

## 4. 模块生命周期

| 状态 | 定义 |
|---|---|
| `planned` | 属于目标范围，尚未进入开发 |
| `developing` | 正在设计、开发或完善 |
| `uat_pending` | 已具备候选版本，待进入 UAT |
| `uat_in_progress` | 正在执行当前版本 UAT |
| `uat_passed` | 当前版本已通过约定 UAT |
| `limited_release` | 仅向指定项目、租户或角色开放 |
| `blocked` | 存在阻断性问题 |
| `production_enabled` | 未来已在真实生产环境正式开放 |

模块可在 `uat_passed` 后进入限制开放，也可因版本变更重新回到 `uat_pending`。历史 PASS 不自动代表当前提交仍通过。

## 5. 近期产品演进

后续房产经营业务按以下顺序推进：

1. 校正产品范围与环境定义。
2. 建立共享房产底座和长短租互斥占用能力。
3. 实施整套房源民宿 MVP。
4. 实施集中式公寓/人才公寓住房出租 MVP。
5. 建立园区商业租赁、民宿和住房出租的跨业态经营分析。

领域规划见 Trellis 任务 `07-24-property-business-domain-planning`。

## 6. 分批 UAT 与开放

- 每批必须明确模块、角色、业务流程、测试数据和清理方案。
- UAT PASS 必须记录对应 commit、环境、执行时间和证据。
- 限制开放必须同时配置模块授权、菜单、角色权限和数据范围。
- 跨模块依赖未通过时，不能仅凭页面可访问判定模块通过。
- 财务、认证、迁移、文件和外部集成模块继续执行更严格的专项门禁。
- 正式投产前另行完成独立的 Production Go/No-Go，不复用 UAT PASS 直接替代。

## 7. 历史文档解释

- `first-release-*` 脚本和文档名称属于历史命名，仍可作为核心回归入口。
- `production-gate`、`production-hardening` 等报告记录的是生产相似配置或目标环境演练，不代表系统已产生真实生产运营事实。
- 历史菜单白名单表示当时的展示/验收集合，不表示其它模块已从产品范围删除。
- 历史报告不得为适配当前口径而篡改原始结果；需要校正时，应在当前权威入口或新报告中追加说明。
