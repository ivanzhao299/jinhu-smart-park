# S1 Test Plan

S1 覆盖组织、用户、角色、权限、字典、附件、审计。测试优先验证横切约束，再验证业务 CRUD。

## API 横切测试

- 未登录访问 `/api/v1/users/me` 返回 401。
- 普通用户无权限访问系统管理接口返回 403。
- 超级管理员角色 `SUPER_ADMIN` 可访问所有权限点。
- 权限 `*` 可访问所有权限点。
- 写请求不携带 `X-Idempotency-Key` 返回 400。
- 写请求成功写入 `sys_op_log`。
- 写请求失败也写入 `sys_op_log` 且 `success = false`。
- 列表接口忽略前端传入的 `tenant_id`、`park_id`，只使用 JWT 上下文。
- 删除接口只更新 `is_deleted`，不物理删除。

## 组织测试

- 创建组织时 `org_code` 在同一 `tenant_id + park_id` 下唯一。
- 组织列表默认过滤 `is_deleted = false`。
- 组织树按 `parent_id` 组装。
- 停用组织后不能作为新增用户主组织。

## 用户测试

- 创建用户时密码使用 bcrypt 存储，不保存明文。
- 用户列表支持 `page`、`page_size`、`keyword`、`status`、`sort`。
- 重置密码必须记录操作日志。
- 分配角色必须记录权限变更操作日志。
- 禁用用户不能登录。

## 角色与权限测试

- 角色列表按租户和园区隔离。
- 角色授权只能绑定存在且未删除的权限点。
- 普通角色只能访问授权接口。
- 删除角色为软删除，不影响历史审计记录。

## 字典测试

- 字典类型编码在同一 `tenant_id + park_id` 下唯一。
- 禁用字典类型后，前端下拉数据不返回该类型字典项。
- 字典项排序稳定。

## 附件测试

- 附件元数据创建必须包含 `biz_type`、`biz_id`、`file_name`、`storage_key`。
- 附件删除为软删除。
- 附件列表按 `biz_type + biz_id` 查询时必须带租户和园区隔离。

## 审计测试

- 登录日志支持按用户名、成功状态、时间范围查询。
- 操作日志支持按模块、动作、用户、成功状态查询。
- 审计日志不可通过普通删除接口物理删除。
