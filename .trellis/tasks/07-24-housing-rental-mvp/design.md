# 住房出租 MVP 技术设计

## 1. 聚合

- `ResidentialTenant`：个人租客。
- `ResidentialCohabitant`：同住人关系。
- `ResidentialLease`：住宅租约。
- `ResidentialLeaseChargePlan`：租金和周期费用计划。
- `ResidentialHandover`：入住/退租交割。
- `ResidentialDepositLedger`：押金收取、抵扣和退款。
- `OperatingPurchase`：轻量采购及成本归集。

## 2. 与现有 leasing 的关系

推荐复用能力而不是直接复用企业合同实体：

- 复用编码、审批、附件、生效、续租、变更和审计模式。
- 复用应收、收款、核销、减免、退款的财务底座。
- 新建住宅租约聚合，避免向 `biz_leasing_contract` 强加个人租客、同住人和住宅交割字段。
- 通过共享占用账本与 `biz_unit` 建立租期排他关系。

## 3. 租约状态

```text
draft → pending_approval → pending_signature → active
active → expiring → renewed
active → checkout_pending → terminated
draft/pending_* → void
```

租约生效必须满足审批通过、签署登记完成、房源可用和占用创建成功。

## 4. 周期账单

- 支持每 1/3/6/12 或自定义月数一期。
- 账期生成必须处理非整月首期、尾期和到期日。
- 租金、物业等固定费用来自租约费用计划。
- 水电燃气可以来自能源表计账期或人工账单。
- 所有应收保存租约、费用项和计费期间来源。

## 5. 押金与交割

- 押金作为独立台账，不混入普通收入。
- 退租时先完成未结费用、损坏赔偿和抵扣，再生成退款。
- 入住/退租交割保存物品、照片、表底、钥匙门卡和附件。

## 6. 采购成本

- 采购单和明细属于内部成本。
- 采购付款状态不等同于租客收款。
- 转收费生成独立租客应收，并保存采购明细来源和分摊关系。
- 不维护库存数量余额。

## 7. UI

- 后台：个人租客、租约、周期账单、交割、采购和退租结算。
- 移动端：现场建档、拍照、表底、物品清单、签署件上传和退租验收。
- 敏感字段、金额和高风险动作按权限显示。

## 8. 安全

- 个人身份数据加密/脱敏。
- 生效、周期账单生成、收退款和转收费具备幂等或唯一约束。
- 财务删除保持软删除/作废/冲销语义。
