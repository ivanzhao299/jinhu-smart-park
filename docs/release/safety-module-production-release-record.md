# 安全巡检与隐患整改模块生产发布归档

## 1. 发布结论

安全巡检与隐患整改核心模块已完成生产发布。

本次发布仅代表安全巡检与隐患整改核心菜单已开放，不代表所有安全管理能力均已开放。安全现场执行专属入口菜单、应急、作业许可、移动端、通知、AI、大屏、复杂审批流、机器人和外部平台对接仍按本文第 3 节暂缓。

本文用于交付归档和后续审计，不包含真实生产 IP、域名、账号、密码、密钥、Token 或敏感连接串。

本次生产实际开放以下安全核心路径：

- `/safety/dashboard`
- `/safety/inspect-points`
- `/safety/inspect-templates`
- `/safety/inspect-plans`

## 8. 回滚策略

1. 菜单入口回滚
   - 从 `FIRST_RELEASE_MENU_PATHS` 移除第 2 节 safety 路径。
   - 保留后端接口、数据库结构和既有业务数据。

2. 权限回滚
   - 撤销普通角色 safety 菜单和接口权限。
   - 保留必要管理员权限用于历史数据查询和应急处理。

3. 模块授权回滚
   - 如需完全隐藏，关闭租户 `safety` 模块授权。
   - 回滚前确认不会影响已有审计和历史数据查询。

4. 数据保留原则
   - 不直接删除生产巡检点、巡检模板、巡检计划、巡检任务、隐患、状态日志、附件和审计数据。
   - 如默认模板被判定不适合生产，应通过后续安全迁移停用或替换，不手工删除历史记录。
