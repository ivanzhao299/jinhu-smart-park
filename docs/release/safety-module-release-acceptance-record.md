## 9. 回滚策略

优先采用入口回滚，避免破坏已产生的生产数据。

1. 入口回滚
   - 从 `FIRST_RELEASE_MENU_PATHS` 移除 safety 核心路径。
   - 保留后端接口、数据库结构和历史业务数据。

2. 权限回滚
   - 撤销普通角色 safety 菜单和接口权限。
   - 保留必要管理员权限用于历史数据查询和应急处理。

3. 模块授权回滚
   - 如需完全隐藏，关闭租户 `safety` 模块授权。
   - 回滚前确认不会影响已有巡检、隐患和审计数据查询。

4. 数据保留原则
   - 不直接删除生产巡检点、模板、计划、任务、隐患、状态日志、附件和审计记录。
   - 如默认模板被判定不适合生产，应通过后续安全迁移停用或替换，不手工删除历史记录。
