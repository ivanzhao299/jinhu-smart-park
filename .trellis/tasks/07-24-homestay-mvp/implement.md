# 民宿 MVP 实施计划

## 1. 批次

### A. 房态与价格

- [ ] 民宿模块、权限和共享契约。
- [ ] 基础日价、日期覆盖和价格日历。
- [ ] 统一占用接入和房态读取模型。

### B. 预订

- [ ] 订单、逐夜价格、来源和取消规则快照。
- [ ] 创建、确认、取消、改期和 No-show。
- [ ] 并发锁房与幂等测试。

### C. 入住退房

- [ ] 多入住人实名登记。
- [ ] 凭证发放/回收。
- [ ] 入住、续住、退房、扣费和退款登记。

### D. 保洁周转

- [ ] 退房自动创建保洁任务。
- [ ] 移动端拍照、耗材和异常上报。
- [ ] 检查/可售恢复和维修联动。

### E. 报表和 UAT

- [ ] 今日到离店、入住率、平均房价、收入和待保洁。
- [ ] 管理后台与 390px 移动端验收。
- [ ] 完整预订到房态恢复 E2E。

## 2. 回滚点

- 每批由独立权限和模块授权控制。
- 新表和新路由不改变现有 leasing 行为。
- 关闭民宿模块后，资产和商业租赁继续工作。
- 不在回滚中删除订单或财务记录。

## 3. 必测场景

- 同房源同夜并发确认。
- 连住边界、同日退房再入住。
- 取消释放、改期冲突、No-show。
- 金额快照、人工退款和特殊减免。
- 证件权限和脱敏。
- 退房后保洁未完成不得可售。
