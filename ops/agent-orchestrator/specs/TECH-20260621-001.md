## Queue Semantics

- All tasks start with `status: READY`.
- Each agent claims its own READY task with:
  ```bash
  node ops/agent-orchestrator/scripts/claim-task.mjs agent-2
  ```
- Claiming updates the task to `CLAIMED` and writes a lock.
- Agent completion is recorded with:
  ```bash
  node ops/agent-orchestrator/scripts/complete-task.mjs --result /path/to/result.json
  ```
- The orchestrator audits changed-file boundaries with:
  ```bash
  node ops/agent-orchestrator/scripts/audit-agent-result.mjs <task_id>
  ```
